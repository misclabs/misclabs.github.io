Responsive Design Notebook

History

First there was fixed-width layouts. Then liquid layouts that had more fexibility, but failed at the extrems. Then when media queries first came along there were adaptive layouts that switched between a few fixed width layouts.

Finally responsive designs used media queries to switch between a few liquid designs. It was defined with three criteria:

  1. Fluid grids
  2. Fluid media
  3. Media queries

There was a problem where browsers assumed pages were designed with 980 pixel width in mind, and the browser would apply a 980 width and scale the rendered page down to the actual width of the screen. A meta element in the head of the page can tell the browser not to do this.

<meta name="viewport" content="width=device-width, initial-scale=1">

width=device-width tells the browser to assume the width of the website is the same as the device. initial-scale=1 tells the browser not to scale the page.

Media Queries

Media queries are used, among other things, to adjust to a devices form factor: width, aspect ratio, etc. They are CSS rules defined with the @media keyword. Media queries can also be used to adjust layout for printing

Media queries can be specified in CSS or on the CSS's link element.

body { color: black; background-color: grey; } @media print { body { background-color: transparent; } }

<link rel="stylesheet" href="global.css"> <link rel="stylesheet" href="print.css" media="print"> If the media attribute isn't specified it defaults to all

Media Query Features

Conditions can be added to media quieries with the syntax: @media type and (feature). The rules are only applied if the type and features match. Conditions can also be applied in link tag's media attribute. The point at which a media feature condition becomes true is called a breakpoint

Orientation

@media all and (orientation: landscape) { /* Rules... */ } @media (orientation: portrait) { /* Rules... */ } Orientation just refers to the aspect ratio of the screen. The all and is optional since the all is default. Unless you want to specify screens only: @media screen and...

CSS units

em
Calculated font-size of an element.
ch
Width of 0 glyph. ~0.5em
px
One pixel.
in
One inch. 1in = 96px
pt
One point. 1pt = 1/72nd of 1in

For low-dpi devices px represents the physical reference pixel. 1in=96px=72pt, but dimensions in my be different than the physical measurment.

For high-dpi devices in are the same as the physical unit and px is defined as 1/96 of 1in

Viewport Size

@media (min-width: 400px) { // Styles for viewports wider than 400 pixels. }

@media (max-width: 400px) { // Styles for viewports narrower than 400 pixels. }
